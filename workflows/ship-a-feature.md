# Workflow: Ship a Feature

## Goal
Deliver a scoped feature from requirement to production with quality and traceability.

## Best For
Incremental product improvements and roadmap commitments.

## Required Inputs
Feature request, user problem statement, acceptance criteria, technical constraints.

## Participating AI Employees
PM, UX, Architect, Fullstack, QA, Debugger, DevOps.

## Step-by-Step Execution
1. Validate problem and define acceptance criteria.
2. Produce implementation and test plan.
3. Build feature incrementally.
4. Execute QA and bug resolution cycle.
5. Run deployment checklist and release.
6. Monitor post-release performance and incidents.

## Founder Approval Gates
- Approve final feature scope.
- Approve release candidate after QA signoff.
- Approve production deployment.

## Expected Outputs
Feature spec, implementation notes, QA report, release notes, incident watchlist.

## Common Failure Modes
Ambiguous acceptance criteria, integration blind spots, rushed release decisions.

## Success Metrics
Cycle time, escaped defects, rollback rate, feature adoption.
