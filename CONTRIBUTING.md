# Contributing

Thanks for helping improve SoloCorp AI OS.

## Contribution principles
- Keep the project **framework-agnostic**.
- Prefer **practical operating docs** over hype.
- Optimize for solo founders executing real work.
- Preserve founder approvals for high-risk actions.

## How to contribute
1. Open an issue describing the problem and expected outcome.
2. Propose changes in docs-first form (role spec, workflow, template, or schema).
3. Keep pull requests focused and easy to review.
4. Add or update documentation when behavior changes.

## Content standards
- Markdown-first, readable in plain text.
- Use clear sections and checklists.
- Avoid personality-heavy role prompts.
- Include guardrails and approval requirements where relevant.

## Pull request checklist
- [ ] Content is original and not copied from other repos.
- [ ] Links are valid and relative paths resolve.
- [ ] New role/workflow files follow required format.
- [ ] Changes improve practical execution quality.

## Development notes
No custom runtime is required. Validate by checking markdown quality and structure consistency.
