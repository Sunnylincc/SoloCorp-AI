# Handoff Rules

## Handoff Packet (required)
1. Context summary
2. Goal and success criteria
3. Inputs and source links
4. Constraints and deadlines
5. Open questions
6. Requested output format

## Quality Standard
- No handoff without explicit owner and due date.
- Include assumptions and known risks.
- Avoid forwarding raw notes without synthesis.

## Example Pattern
- PM Agent -> Architect Agent for technical design
- Architect Agent -> Fullstack Agent for implementation plan
- Fullstack Agent -> QA Agent for validation and release readiness
